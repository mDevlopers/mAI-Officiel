## 2024-04-12 - ARIA Labels on prominent Icon-Only Buttons\n**Learning:** Found several highly prominent icon-only UI interactions (Artifact Close, Context Actions, Dictation, Stop Generation) lacking ARIA labels, creating significant accessibility issues for screen reader users trying to interact with core chat functionalities.\n**Action:** Always verify that every icon-only `<button>` or `<Button>` component includes a descriptive `aria-label`, especially in dynamic components where the label might need to reflect state changes (e.g., `isRecording`).
## 2024-06-18 - Missing ARIA Labels on Icon-only Buttons
**Learning:** Found an accessibility issue pattern where `size="icon"` buttons within the application (e.g., in `AgentListDialog`) often lacked descriptive `aria-label` attributes. This makes screen readers unable to convey the button`s purpose to users.
**Action:** Always ensure that icon-only `<button>` or `<Button>` components include a descriptive `aria-label`. Checked `components/chat/agent-list.tsx` specifically today and addressed missing attributes for edit and delete actions.
