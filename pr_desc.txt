🎯 **What:** The testing gap addressed
The file `lib/notifications.ts` handles storing and retrieving notifications from `window.localStorage` and dispatching custom events, but its logic wasn't fully tested. This PR adds a global mock for the `window` environment to correctly test these CRUD and notification methods.

📊 **Coverage:** What scenarios are now tested
- `getNotificationHistory`: Empty state, valid array, and max element capping.
- `createNotification`: Creating events, setting default title, saving correctly, and dispatching events.
- `createAiResponseNotification`: Handling `started`, `completed`, and `error` phases and message template mapping.
- `markNotificationRead` & `markAllNotificationsRead`: Marking specific and all items as read correctly.
- `clearNotifications`: Correctly deleting stored notifications.
- `subscribeNotifications`: Subscribing and unsubscribing from event listener successfully.

✨ **Result:** The improvement in test coverage
The previously untested core logic of the client-side notification module (`lib/notifications.ts`) is now rigorously covered under unit testing using an emulated DOM strategy, preventing regressions in future refactors.
